The Supervised Learning Problem¶

Standard working assumption in supervised learning is that the mechanish that generates the data can be desribed as

$$ Y = f(X) + \epsilon $$

$f$ is a function, $X=(X_1,X_2,\dots,X_p)$ is a vector of inputs/features, $Y$ is the labels/output/response and $\epsilon$ is random noise.

General Problem:¶

We are interested in estimating the function $f$ using a dataset consisting of labels: $Y$ and features: $X$, that are generated by the same mechanish described above.

Machine Learning¶

Creating a pyspark.ml model is very similar to the standard sklearn API but there are some differences.

  1. You can specify the subset of input features in a pipeline.
  2. When predicting you use a .transform() method on a fitted model object, which adds extra column(s) based on the model.

Example: Dataset¶

The california_housing dataset contains:

  • MedHouseVal Median house value for California districts, expressed in hundreds of thousands of dollars ($100,000).
  • MedInc Median income in block group
  • HouseAge Median house age in block group
  • AveRooms Average number of rooms per household
  • AveBedrms Average number of bedrooms per household
  • Population Block group population
  • AveOccup Average number of household members
  • Latitude Block group latitude
  • Longitude Block group longitude

Classification and Regression Trees (CART)¶

We choose the variable and split-point to achieve the best fit. Then one or both of these regions are split into two more regions, and this process is continued, until some stopping rule is applied.

Fitting a DecisionTreeRegressor¶

In [4]:
from pyspark.ml import Pipeline
from pyspark.ml import feature
from pyspark.ml import regression

# Specify the response variable and the input features.
RESPONSE = 'MedHouseVal'
FEATURES = ['MedInc', 'HouseAge', 'AveRooms', 'AveBedrms', 'Population', 'AveOccup', 'Latitude', 'Longitude']

# Stages in the pipeline
assembler = feature.VectorAssembler(
  inputCols=FEATURES,
  outputCol='features'
)
dt = regression.DecisionTreeRegressor(
    labelCol=RESPONSE, 
    maxDepth=3, 
    minInstancesPerNode=256,
)
pipeline = Pipeline(stages=[assembler, dt])

# Fit the model to the training data.
model = pipeline.fit(train)

Grid Search¶

Important to set a seed again.

In [5]:
from pyspark.ml import tuning
from pyspark.ml import evaluation

# A seed for the random sampler, used for reproducibility.
SEED = 42

estimator_param_maps = (
    tuning.ParamGridBuilder()
    .addGrid(dt.maxDepth, [2, 4, 8])                  # Max depth of tree (maximum 2 ** maxDepth leaf nodes).
    .addGrid(dt.minInstancesPerNode, [32, 64, 128])   # Minimum number of observations in a leaf node.
    .build()
)

evaluator = evaluation.RegressionEvaluator(
    labelCol=RESPONSE, 
    predictionCol="prediction", 
    metricName="rmse"
)

cv = tuning.CrossValidator(
    estimator=pipeline, 
    evaluator=evaluator, 
    estimatorParamMaps=estimator_param_maps, 
    numFolds=5, 
    seed=SEED
)
cvModel = cv.fit(train)

In-Sample and Out-of-Sample Error¶

The transform() method in all pyspark.ml models adds a column called prediction. This is different compared to the standard sklearn API.

We see that it does worse on the test dataest. This is to be expected.

In [10]:
from pyspark.sql import functions as F

display(
  model.transform(train)
  .withColumn(
    'residual',
    F.col('prediction') - F.col(RESPONSE)
  )
  .agg(
    F.mean(F.abs(F.col('residual'))).alias('mean_absolute_error'),
    F.mean(F.pow(F.col('residual'), 2)).alias('mean_squared_error'),
    F.stddev_pop(F.col('residual')).alias('standard_deviation'),
  )
)

Saving and Loading Models¶

When you load your saved models, you need to specify the type of model you are loading back. It is therefore recommended that you always put your transformers/estimators into a Pipeline s.t. all your models can be loaded with a PipelineModel and you only need to change the file path to the model.

In [15]:
from pyspark.ml import PipelineModel

# Save the model for later use.
model.write().overwrite().save('/tmp/my_great_model')

# Load the model later.
model = PipelineModel.load('/tmp/my_great_model')
